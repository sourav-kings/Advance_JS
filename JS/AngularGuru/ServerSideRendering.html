Server Side Rendering


Why do we need it?

  • Performance benefit for our customers
  • Consistent SEO performance


Advantage over Client side rendering

  1. In CSR, webpage is not visible until all libraries are downloaded and until page rendering is over on browser.
    In SSR, webpage is displayed almost immediately on browser.
    How?
    • Browser will start rendering the HTML from your server without having to wait for all the JavaScript to be downloaded and executed.
    • JS framework lib will be downloaded in both cases:-
      In CSR, until rendering gets over and created virtual dom has moved over to browser dom an empty web-page is displayed.
      However in SSR, the page starts showing up while all of that is happening.
  2. Another Advantage - In SSR there is blank white page flickering until content is fully renderd.
    This issue can be avoided in CSR by using a loading image.


Drawbacks

  • User interaction may fail -
    Until page rendering is complete with all events binded.
  • Time To First Byte slower -
    SSR server side HTML creation is slower than CSR quick empty response
  • Server throughput is very less.

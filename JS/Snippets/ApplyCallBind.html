Apply, Bind, Call

  • All are nearly identical and are used for borrowing methods and for setting the 
    this
    value explicitly 
  • Apply
    and 
    Call
    introduced in ES3. Whereas 
    Bind
    is new and introduced in ES5.
  • this
    does NOT belong to the object to the function containg, instead it belongs to the object which has invoked the function containing 
    this
    .
  • Use 
    Bind
    to explicitly mention which object should the 
    this
    belong to.
    Example:- 
    $ ("button").click (user.clickHandler.bind (user));
  • Another way of explicitly assign object to 
    this
    is to assign the containing function to a variable declared in an another scope. This issue can also be fixed using 
    Bind
  • Borrowing of methods:- 
    Bind
    can be used for that but with a risk of accidental over-write. So better to use 
    Apply
    or 
    Call
  • Diff b\w 
    Apply
    and 
    Call
     lies in the way we pass parameters. In Call we pass parameters one by one, and in Apply we pass parameters in an array.

Function Currying

  • Acheived using 
    Bind
    .
  • Purpose is to create a new variable for the above function call with some preset values. 
    Bind
    .
  • The first parameter in the Bind is always the object which we need to 
    Bind
    the current function with.
  • For ex. for this function 
    function greet (gender, age, name) {
    // function defination
}
    The below code creates a function greetAYoungster variable which would be a function call to the greet function but with preset values for gender and age as "" and 16 resp. The last parameter has not been set.
    var greetAYoungster = greet.bind (null, "", 16);
greetAYoungster ("Alex");​

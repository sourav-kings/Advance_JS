Instance for JS and strongly typed languages?

  • In class based or strong-typed languages, instance has a special meaning and is fundamentally differnet from class.
  • In JS, instance is just an informal way of denoting an object created using constructor function.

References Mozilla Developer

Internal property [[prototype]]

  • Its value is implicitly set when a new object is created using "new" operator.
  • This property allows to acccess the objects parent in the prototype chain. 
    var sourav = new Employee();
    For example in above code, the newly created object 
    sourav
    's internal property is set with 
    Employee.prototype
    property.
  • In modern browsers, we can access this by "__proto__" property.
  • The internal [[Prototype]] property determines the prototype chain used to return property values for this new object. While looking for a property JS checks the current object and when not found keep looking up in the prototype chain until found or last parent.

new operator

  • In the below code, the new operator creates a new generic object and implicitly sets the value of the internal property [[Prototype]] to the value of
